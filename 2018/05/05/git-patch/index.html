git patch

Posted on 2018-05-05 | In 版本控制工具

简介

    当我们对一个文件进行修改之后，我们想要别人来进行打patch的操作，那么我们可以使用git 来对文件进行操作生成patch文件，来让别人方便的对源文件进行打patch操作。

步骤

  1. 首先我们在本地对目标文件进行修改，然后进行git commit提交。
  2. 使用git log 查看当前最新提交的ID和上一次提交的ID。
  3. 生成patch文件
  4. 发给他人，他人对该patch文件进行使用

生成patch

  • 生成带有详细commit信息的patch
    git format-patch -1 commit
  • 生成普通patch文件

使用patch

    在patch对应的Git库下，使用命令合并patch。

# git Patch
