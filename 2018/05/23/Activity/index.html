Activity

Posted on 2018-05-23 | In Android

1 概念

    activity是Android中面向用户的操作界面，对于一个APP来说，会有多个activity来组成，此时多个activity组成activity栈，当前展现或活动的activity处于栈顶。

2 结构

    一个普通的activity可以直接继承自activity，但是如果要实现一些其他特殊的UI显示则可以继承它的子类。

3 配置activity

    Android应用要求所有应用程序组件都要显示的进行配置。
组件包括activity，Service、ContentProvider、BroadcastReceiver。
在AndroidManifest.xml中进行配置。
配置格式如下：

    name: 该activity对应的类的名字，记得前边需要加上一个“.”

    icon : 该activity对应的图标

    label：如果这个activity是启动activity（即第一个执行的activity），那么label对应的名字即为app名字，否则则是label显示的文字

    exported：如果为true的话则该activity可以被其他应用调用

    launchMode：指定该activity的启动方式，一共有四种加载模式（standard、singleTop、singleTask、singleInstance）

    如果配置了filter，那么可以指定该activity作为应用程序的入口。

4 函数方法

    该方法在ACtivity创建的时候被回调

onCreate(Bundle status){
    该函数中可以使用setContentView(View view)方法来展示组件View。
    如果只是简单的处理逻辑功能则不需要显示。
}

    启动ACtivity的方法

startActivity(Intent intent) //  启动其他ACtivity
startActivityForResult(INtent intent,int requestCode)  // 指定的请求码启动Activity，程序会获取新启动的ACtivity返回的结果
startActivityAsUser(...)  //这个方法类似于第一种方法，只是这种方法多了一个参数，即用户ID，这个用户ID可以识别当前用户是否具有相对应的权限。

    关闭ACtivity的方法

finish()    //结束当前的ACtivity
finishActivity(int requestCode)    //结束以startACtivityForResult（）启动的Activity。eInstance）

    配置intent-filter

如果配置了filter，那么可以指定该activity作为应用程序的入口。
