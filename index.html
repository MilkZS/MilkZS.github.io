Type-V

Posted on 2018-05-29 | In NFC

1 名词解释

NCI：NFC Controller Interface

ACM：Active Communication Mode

P2P：peer-to-peer

2 前言

    Type-V是一种新的TAG 技术，于2015年推出。Type-V提供了一种单一的通信模式，Type-V技术是基于RF技术的，符合ISO/IEC15693标准，可以和其他实现了ISO/IEC 15693规范的记忆性TAG很好的兼容。

    Type-V是RF技术支持的第四种NFC规范，（前三种规范分别是Type A，Type B，Type F）。

3 Type-V经常使用的地方

3.1 Library books, products, and packaging

    图书馆中的图书一般都会含有ISO/IEC15693标签，这就意味着如果在TAG里有关于这本图书的信息，那么持有相关APP的NFC devices就会很方便的获取图书的相关信息，提升了图书管理的效率，也方便了读者的阅读。同样这种成本很低。一些商场或某些产品都会有ISO/IEC15693标签，所以它的使用范围很广。

3.2 tickets

    在tickets上边添加ISO/IEC15693标签，TAG里可以添加用户信息，用户可以持有这种tickets进行入场或者旅行，并且这种TAG可以重复使用，只需要修改TAG里的信息即可。

3.3 Healthcare

    一些贴身的医疗设备也含有这种ISO/IEC15693TAG，我们可以使用NFC devices无需链接的随时接触设备从而获取我们身体的相关状况。

4 NCI 2.0特性

4.1 Autonomous tag emulation

    这个特性最大的亮点就是，在NFC devices并没有开机的情况下，仍然可以使用外部读卡器对手机内部模拟的TAG存储的信息进行读取。在之前的NFC 卡模拟中，我们实现卡模拟是通过跑在APP进程中的APP来实现的，如果想要使用这种功能，则必须保证NFC devices或者这个APP进程处于活跃状态，也就是开启状态，否则便不能使用这种功能。而使用Type V规范可以很好的解决这个问题，可以随时使用这种功能。

4.2 RF Interface Extension

    RF 扩展接口是NCI2.0 提出的一个新的概念，它可以优化并加速NFCC和APP进程之间的通信而不需要一个NFC devices。

    这个概念有两个目的：

  • 在NFCC和主进程之间，这种概念促进了PDUs的自主交换，持续保持了LLCP的链接而不需要额外的电源。
  • 激活了和NFC规范TAG进行通信的聚合模式，这样可以在一个单一的NCI信息中传输一批RF指令，在NFC Devices 和NFC 规范TAG进行通信过程中，它阻止了耗时任务在APP进程中的转变，从而提升了通信效率。

4.3 NDEF RF interface

    新的NDEF RF接口使得整合NFC 规范技术到一个嵌入式设备变的更加容易，开发系统也变的更加简单。下层的硬件芯片为上层提供接口，上层不需要过多的关注下层，方便了开发者的开发与维护。

4.4 LLCP v1.3

    新技术使得我们可以为我们的个人或私有信息添加一个未经认证的安全元件选项，使得我们可以在每个设备之间传输我们的私有数据。

1. NFC的运行模式

1.1 NFC包含三种主要的运行模式

  • R/W模式

        该模式交互是单向的。工作模式是具有NFC功能的设备（NFC Device）和NFC TAG之间进行的交互。但是只能由NFC向NFC TAG进行读取和写入操作，例如通过具有NFC功能的手机向北京公交地铁卡充值一样，只能由手机对卡进行查看余额，或者用手机对卡进行充值，但是不能由卡对手机进行读写操作。

        NFC TAG有四种类型，由不同的厂家推出。对应的四种芯片名分别是Topaz（ISO 14443

    14443 TypeA TypeB）。
